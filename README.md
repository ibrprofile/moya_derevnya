# Информационный сайт о малочисленных населённых пунктах РФ

Веб-приложение для каталогизации, управления и просмотра информации о малочисленных населённых пунктах Российской Федерации, включая исторические данные, фотографии и географические сведения.

## Описание проекта

Проект представляет собой информационную систему, которая позволяет:
- Просматривать каталог малочисленных населённых пунктов
- Организовывать данные по географическому принципу (губернии, уезды, поселения)
- Загружать и управлять фотографиями поселений
- Управлять содержимым через удобную админ-панель
- Получать доступ к данным через REST API

## Технический стек

- **Backend**: PHP 7+ с использованием PDO для работы с БД
- **Database**: MySQL / MariaDB
- **Frontend**: HTML5, CSS3, JavaScript (vanilla JS)
- **Архитектура**: MVC-подобная структура с разделением на публичный интерфейс, API и админ-панель

## Требования

- PHP 7.4+
- MySQL 5.7+ / MariaDB 10.3+
- Веб-сервер (Apache с модулем mod_rewrite или Nginx)

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/ibrprofile/moya_derevnya

```

### 2. Настройка базы данных

Отредактируйте параметры подключения в файле [config/database.php](config/database.php):

```php
define('DB_HOST', 'localhost');
define('DB_USER', 'ваш_пользователь');
define('DB_PASS', 'ваш_пароль');
define('DB_NAME', 'ваша_база_данных');
```

Создайте базу данных и импортируйте схему:

```bash
mysql -u ваш_пользователь -p ваша_база_данных < database/schema.sql
```

Опционально импортируйте начальные данные:
```bash
mysql -u ваш_пользователь -p ваша_база_данных < database/seed.sql
```

### 3. Настройка веб-сервера

#### Apache
Убедитесь, что включен модуль `mod_rewrite`. Убедитесь, что `.htaccess` файлы разрешены в конфигурации VirtualHost:

```apache
<Directory /путь/к/проекту>
    AllowOverride All
    Require all granted
</Directory>
```

#### Nginx
Создайте конфигурацию сервера:

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/project;
    index index.html index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}
```

### 4. Установка прав доступа

```bash
chmod 755 admin/
chmod 755 -R uploads/    # если директория существует
```

## Использование

### Публичный интерфейс

- **Главная страница**: `/index.html` — информация о проекте
- **Каталог**: `/catalog.html` — просмотр каталога поселений с фильтрацией
- **Детали поселения**: `/settlement.html?id=<id>` — полная информация о поселении

### Админ-панель

Доступ по адресу `/admin/index.php`

**Функции:**
- Авторизация администраторов
- Управление категориями содержимого
- Добавление и редактирование поселений
- Загрузка и удаление фотографий
- Управление географической иерархией (губернии → уезды → поселения)
- Просмотр статистики

### REST API

API предоставляет доступ к данным в JSON формате:

- `GET /api/settlements.php` — список всех поселений
- `GET /api/settlement-details.php?id=<id>` — детали поселения
- `GET /api/categories.php` — список категорий
- `GET /api/search.php?q=<query>` — поиск поселений
- `GET /api/gubernii.php` — список губерний
- `GET /api/regions.php` — список регионов
- `GET /api/uezds.php` — список уездов
- `GET /api/stans.php` — список станций

## Основные возможности

### Каталогизация данных
- Иерархическая организация поселений по географическому принципу
- Поддержка метаданных (год основания, координаты, топоним, период деятельности)
- Статусы поселений (активное/неактивное)

### Галерея фотографий
- Загрузка фотографий для каждого поселения
- Управление галереей через админ-панель
- Отображение фотографий в каталоге

### Поиск
- Полнотекстовый поиск по названиям поселений
- Фильтрация по категориям и географическим регионам
- Навигация через breadcrumb

### Многоязычная поддержка
- Интерфейс на русском языке
- UTF-8 кодировка для корректной работы с кириллицей

## Миграции и обновления

Проект включает скрипты миграции для обновления схемы БД:

- `database/migration-v2.sql` — обновление до версии 2
- `database/add-gubernii-uezd-stan.sql` — добавление данных о губерниях /уездах
- `database/update-schema-add-custom-status.sql` — добавление пользовательских статусов

Для применения миграции:
```bash
mysql -u пользователь -p база < database/migration-v2.sql
```

## Безопасность

⚠️ **Важно для production:**

1. Изменить учетные данные базы данных в [config/database.php](config/database.php)
2. Реализовать аутентификацию в админ-панели (переменные сессии, хеширование паролей)
3. Использовать HTTPS
4. Ограничить прямой доступ к файлам конфигурации через `.htaccess` (Apache):
   ```apache
   <FilesMatch "\.php$">
       Require all denied
   </FilesMatch>
   ```
5. Валидировать и санитизировать все входные данные
6. Использовать prepared statements для всех SQL запросов
7. Периодически обновлять версии компонентов
